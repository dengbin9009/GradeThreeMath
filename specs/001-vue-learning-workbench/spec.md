# Feature Specification: Vue 数学母题学习工作台重构

**Feature Branch**: `001-vue-learning-workbench`

**Created**: 2026-06-22

**Status**: Draft

**Input**: 将现有上海三年级数学提升项目用 Vue 重构，优化 UI，突出题目主题，把必要选项放入可筛选的侧边栏或上边栏，并提升交互的人性化程度；增加账号鉴权，区分管理员和用户，管理员可以添加用户并管理用户有效时间。

## Clarifications

### Session 2026-06-22

- Q: 账号系统是否保留在本次重构范围？ → A: 保留，系统使用必须鉴权，管理员可添加用户并管理有效时间。
- Q: 领域迁移任务是否严格测试先行？ → A: 是，重排领域测试并将批量迁移测试与实现拆开。
- Q: 任务如何满足可追踪要求？ → A: 每个任务显式补充关联的 FR 与 SC 编号，其余分析建议全部采纳。

### Session 2026-06-24

- 动画精修方向：逐个母题优化为精致、有趣、轻松好笑的大舞台数学动画；笑点必须服务数学状态，不做无关装饰。
- 图片资产方式：精修图片统一使用 `image2` 规划和生成，按背景、可动对象、状态帧、数学道具、反馈道具和 fallback 分层；图片内不写可变数字、题干、公式或答案。

## User Scenarios & Testing

### User Story 1 - 通过有效账号进入系统 (Priority: P1)

用户使用管理员分配的登录名和密码进入系统。只有处于启用状态且当前时间位于账号有效期内的用户可以使用学习功能；首次登录必须修改临时密码。

**Why this priority**: 鉴权是全部学习和管理功能的入口，未通过鉴权时其他用户故事不可用。

**Independent Test**: 分别使用有效、未生效、已到期、已停用和错误密码账号登录，只有有效账号能进入母题总览。

**Acceptance Scenarios**:

1. **Given** 有效普通用户，**When** 输入正确登录名和密码，**Then** 创建安全会话并进入原目标页面。
2. **Given** 首次登录用户，**When** 登录成功，**Then** 必须先修改临时密码，完成前不能访问学习页。
3. **Given** 账号尚未到生效时间、已到期、已停用或凭证错误，**When** 尝试登录，**Then** 不创建会话并统一显示“登录名、密码或账号状态不可用”，不得泄露账号是否存在或具体状态。
4. **Given** 已登录账号被管理员停用或缩短有效期至当前时间之前，**When** 再次请求受保护资源，**Then** 服务端拒绝请求、撤销会话并回到登录页。
5. **Given** 用户已登录，**When** 主动退出，**Then** 服务端会话失效且浏览器不能通过后退继续使用受保护功能。

---

### User Story 2 - 管理员管理用户和有效期 (Priority: P1)

管理员可以在用户管理页面添加普通用户、设置登录名和临时密码、选择账号生效时间与到期时间，并能搜索、暂停、恢复、延长有效期、重置密码和强制下线。

**Why this priority**: 没有管理员维护账号和有效期，封闭鉴权系统无法持续使用。

**Independent Test**: 管理员创建一个明天生效、30 天后到期的用户，修改为立即生效，用户完成首次登录；随后管理员停用该用户，现有会话立即失效。

**Acceptance Scenarios**:

1. **Given** 管理员进入用户管理，**When** 创建普通用户并设置有效期，**Then** 系统生成唯一账号并返回一次性临时密码确认。
2. **Given** 普通用户，**When** 直接访问管理路由或管理 API，**Then** 服务端返回无权限，前端不展示管理内容。
3. **Given** 已到期用户，**When** 管理员延长到期时间并保持启用，**Then** 用户可以重新登录，但旧会话不会恢复。
4. **Given** 活跃用户，**When** 管理员暂停、重置密码或收紧有效期，**Then** 该用户全部会话被撤销。
5. **Given** 管理员修改用户信息，**When** 操作成功，**Then** 审计日志记录操作者、目标用户、动作、时间和非敏感变更摘要。
6. **Given** 当前系统只剩一个管理员，**When** 受控运维流程尝试停用、删除或降级该管理员，**Then** 系统拒绝操作以避免失去管理入口；普通管理员 UI 不提供管理员角色变更入口。

---

### User Story 3 - 孩子专注完成一个母题 (Priority: P1)

孩子打开任意母题后，先看到题目名称、题干、关键条件和大尺寸动画舞台；他可以直接操作动画、观察数量变化、逐步揭示公式并复位，不需要先理解复杂导航。

**Why this priority**: 这是产品的核心价值。重构成功与否首先取决于孩子能否专注理解一道题，而不是页面是否展示了更多信息。

**Independent Test**: 直接访问任一母题 URL，只使用题目区和动画区即可完成一次“读题、操作、检查、复位”的学习闭环。

**Acceptance Scenarios**:

1. **Given** 用户打开 M39，**When** 切换“看现在、撤销还回、撤销借出、正向验算”，**Then** 同一舞台中的书架、借出篮和还回篮数量同步变化，算式与画面一致。
2. **Given** 用户打开任意互动母题，**When** 改变参数，**Then** 题干、动画关键量和答案保持同步且只出现整数。
3. **Given** 用户操作到任意步骤，**When** 点击复位，**Then** 参数、动画、反馈和揭示步骤恢复该题默认状态。
4. **Given** 动画使用图片资产，**When** 数量发生变化，**Then** 图片对象由程序逐个渲染或移动，不以静态整图代替数量变化。
5. **Given** 某母题完成精修，**When** 孩子操作参数或步骤，**Then** image2 分层资产、动态对象、量线和关系式同步变化，且主舞台不退化成题干旁的小图。

---

### User Story 4 - 快速找到适合的母题 (Priority: P1)

家长或孩子可以按知识领域、上下册、课内/拔高、难度、是否有图片动画筛选，并通过搜索或母题编号快速定位内容。筛选器不抢占题目舞台空间。

**Why this priority**: 39 个母题已超过平铺列表的舒适范围，必须降低查找成本，否则内容越完整越难使用。

**Independent Test**: 从全部母题开始，只用筛选和搜索在 10 秒内找到“M20 鸡兔同笼”，然后一键进入学习页。

**Acceptance Scenarios**:

1. **Given** 桌面宽度，**When** 用户展开左侧筛选栏，**Then** 可以多条件组合筛选，中央题目舞台宽度仍满足动画最低尺寸。
2. **Given** 手机宽度，**When** 用户点击筛选按钮，**Then** 筛选项在底部抽屉中出现，应用后自动收起并显示已选条件数量。
3. **Given** 已设置多个筛选条件，**When** 用户清除单个标签或全部清除，**Then** 结果与 URL 状态同步更新。
4. **Given** 搜索“K20”“M20”“鸡兔”或“抬腿”，**When** 输入完成，**Then** 相关知识点与母题都可被定位。

---

### User Story 5 - 在动画中获得及时且不过度的帮助 (Priority: P1)

孩子操作正确时得到简短确认，操作不成立时获得指向关键量的提示，而不是直接弹出答案。每个动画使用统一的操作栏和步骤反馈，但保留题目特有的场景。

**Why this priority**: 动画只有在操作、反馈和数学关系连成一体时才是学习工具，否则只是装饰。

**Independent Test**: 在 M12、M20、M39 各完成一次错误尝试和正确尝试，检查反馈是否指向当前模型且不会遮挡动画。

**Acceptance Scenarios**:

1. **Given** 用户给出不合法参数组合，**When** 系统检测到数学关系不成立，**Then** 阻止无效状态并说明应调整哪个量。
2. **Given** 用户处于第一步，**When** 点击“下一步”，**Then** 只揭示下一层关系，不提前显示最终答案。
3. **Given** 用户启用减少动态效果，**When** 切换步骤，**Then** 状态立即更新且不使用位移动画，信息仍完整。
4. **Given** 精修动画触发轻幽默反馈，**When** 反馈出现，**Then** 它必须绑定当前数学状态、贴近相关对象、短暂显示且不遮挡关键数量。

---

### User Story 6 - 家长获得陪练支持 (Priority: P2)

家长可以从不遮挡动画的陪练抽屉查看讲解话术、常见错因、追问方式和当前子题解题步骤，并可切换基础、变式和挑战子题。

**Why this priority**: 产品目标包含家长陪练，但家长信息不能成为孩子学习时的视觉噪音。

**Independent Test**: 在任一母题打开家长抽屉，完成“查看错因、切换子题、收起抽屉”，动画状态不丢失。

**Acceptance Scenarios**:

1. **Given** 动画正在某一步，**When** 打开和关闭家长抽屉，**Then** 当前参数和动画步骤保持不变。
2. **Given** 一个母题包含多个子题，**When** 切换子题，**Then** 题干、默认参数、提示和答案规则更新，母题导航位置保持不变。
3. **Given** 家长选择“只看提示”，**When** 孩子尝试作答，**Then** 最终答案默认折叠。

---

### User Story 7 - 在不同设备上连续学习 (Priority: P2)

用户可以在桌面、平板和手机上使用完整功能。布局变化时，题目舞台、操作按钮和当前步骤保持可见、可触达且不重叠。

**Why this priority**: 家庭使用设备不固定，现有大动画必须在窄屏上保持可理解，而不是缩成缩略图。

**Independent Test**: 在三个规定视口完成同一母题的筛选、参数调整、步骤切换和家长提示查看。

**Acceptance Scenarios**:

1. **Given** `390x844` 视口，**When** 打开任一母题，**Then** 题目标题、题干、主动画和主要操作均可单列展示，无横向滚动。
2. **Given** `1024x768` 视口，**When** 展开筛选，**Then** 舞台不会被压缩成小图，辅助面板使用抽屉或折叠布局。
3. **Given** 键盘用户，**When** 依次使用 Tab 和方向键，**Then** 可访问筛选、母题列表、动画控制和陪练抽屉，并能看见焦点。

---

### Edge Cases

- 数据文件加载失败时，显示可重试的错误状态，不渲染空白舞台。
- 某母题图片缺失时，使用程序化占位场景并保留全部计算与操作，不显示破图标记。
- 组合筛选无结果时，显示当前筛选摘要和“清除筛选”，不显示空白列表。
- URL 中存在未知 K/M 编号时，回到母题总览并给出轻量提示。
- 参数接近上限时，舞台必须保持稳定；大量对象允许分组显示，但可见计数必须精确。
- 除法、速度、分数应用等母题只能生成整除或整数结果；无合法整数时自动校正参数并提示原因。
- 快速连续拖动或切换步骤时，旧动画必须取消，最终画面与最后一次输入一致。
- 浏览器刷新或前进后退时，当前母题、子题和筛选状态从 URL 恢复；临时动画过程可复位。
- `prefers-reduced-motion` 开启时，不得隐藏任何依赖动画才能理解的信息。
- 登录失败时使用统一错误响应，不能暴露登录名是否存在。
- 登录连续失败时触发限流；限流恢复后有效用户仍可登录。
- 账号 `validFrom` 等于当前时间时立即有效；`validUntil` 等于当前时间时立即失效。
- 服务器和浏览器时区不同时，以服务器 UTC 时间为判定基准，以亚洲/上海时区展示。
- 管理员并发编辑同一用户时，旧版本更新必须被拒绝并提示刷新。
- 用户在多个设备登录后被停用、到期或重置密码时，全部会话必须失效。

## Requirements

### Functional Requirements

- **FR-001**: 系统 MUST 使用主题化学习工作台作为首屏，而不是营销页。
- **FR-002**: 系统 MUST 为每个母题提供稳定、可直接访问的路由，至少包含母题 ID；子题 ID 可选。
- **FR-003**: 系统 MUST 在题目主区域展示母题编号、题目名称、层级、题干、已知条件和待求量。
- **FR-004**: 系统 MUST 将动画舞台作为详情页最大的内容区域，筛选器和陪练信息不得覆盖其核心操作。
- **FR-005**: 系统 MUST 提供知识领域、学期、来源层级、难度、互动状态、图片动画状态筛选。
- **FR-006**: 系统 MUST 支持按知识点名称、母题名称、K/M 编号和关键词搜索。
- **FR-007**: 系统 MUST 将筛选、搜索、当前母题和子题同步到 URL，使刷新和分享后可恢复。
- **FR-008**: 系统 MUST 提供筛选结果数、已选条件标签、单项移除和全部清除。
- **FR-009**: 每个互动母题 MUST 遵循统一控制契约：参数、步骤、提示、检查、复位。
- **FR-010**: 每个互动母题 MUST 保留题目专属场景和对象，不得退化成通用小卡片或纯公式展示。
- **FR-011**: 图片动画 MUST 使用独立对象资源或分层场景，数量由状态动态驱动。
- **FR-012**: 所有生成参数、计算过程和答案 MUST 为整数；系统不得向孩子展示小数。
- **FR-013**: 系统 MUST 在参数变化前后保持题干、舞台数量、计算值和答案规则一致。
- **FR-014**: 系统 MUST 支持逐步揭示，并在默认状态下隐藏最终答案。
- **FR-015**: 系统 MUST 提供不遮挡舞台的正确反馈、纠错提示和边界提示。
- **FR-016**: 系统 MUST 提供家长陪练抽屉，包含话术、常见错因和拓展追问。
- **FR-017**: 系统 MUST 支持同一母题的基础、标准和挑战子题切换，并重置对应动画参数。
- **FR-018**: 系统 MUST 保留现有 38 个知识点、39 个母题、117 个子题及其关联关系。
- **FR-019**: 系统 MUST 在构建时校验蓝图 Schema、ID 引用、动画注册和整数约束。
- **FR-020**: 系统 MUST 提供加载、空结果、缺图、非法参数和运行错误状态。
- **FR-021**: 系统 MUST 支持桌面固定筛选栏、平板可折叠筛选栏和手机底部筛选抽屉。
- **FR-022**: 系统 MUST 支持键盘导航、可见焦点、语义标签和减少动态效果。
- **FR-023**: 系统 MUST 保留现有图片资产的视觉风格，并为缺少合适资产的母题提供一致的场景规范。
- **FR-024**: 系统 MUST 通过统一的母题注册表按需加载动画组件，避免在路由入口硬编码 39 个条件分支。
- **FR-025**: 系统 MUST 在迁移期间提供可对照的计算结果测试，确保 Vue 版本与现有领域逻辑一致。
- **FR-026**: 系统 MUST 禁用公开注册，只允许管理员创建普通用户。
- **FR-027**: 系统 MUST 支持管理员分配的唯一登录名和密码登录，不要求学生提供个人邮箱。
- **FR-028**: 系统 MUST 在所有学习、内容和管理 API 上执行服务端会话鉴权与角色授权；前端路由守卫不能作为唯一授权措施。
- **FR-029**: 系统 MUST 仅支持 `admin` 与 `user` 两种角色，并在服务端强制权限边界。
- **FR-030**: 系统 MUST 为用户保存 `isActive`、`validFrom`、`validUntil` 和 `mustChangePassword`。
- **FR-031**: 系统 MUST 以服务器 UTC 时间判定账号状态：`now < validFrom` 为未生效，`validUntil != null && now >= validUntil` 为已到期。
- **FR-032**: 系统 MUST 允许管理员创建、搜索、分页查看、编辑、暂停、恢复普通用户和管理有效期。
- **FR-033**: 系统 MUST 允许管理员重置用户密码并强制其下次登录修改密码。
- **FR-034**: 系统 MUST 允许管理员撤销用户全部会话；停用、到期、密码重置、角色或有效期收紧时自动撤销会话。
- **FR-035**: 系统 MUST 在每个受保护 API 请求中重新检查用户角色、启用状态和有效期。
- **FR-036**: 系统 MUST 使用 `HttpOnly`、生产环境 `Secure`、`SameSite=Lax` 或更严格的 Cookie 保存会话，不得将凭证存入 `localStorage` 或 `sessionStorage`。
- **FR-037**: 系统 MUST 对登录和敏感管理操作实施限流、来源校验、CSRF 防护和通用错误信息。
- **FR-038**: 系统 MUST 要求管理员创建的临时密码在首次登录后修改，并在修改后撤销其他会话。
- **FR-039**: 系统 MUST 记录用户创建、有效期修改、启停、密码重置、角色变化和会话撤销审计日志，日志不得包含密码或会话令牌。
- **FR-040**: 系统 MUST 防止最后一个管理员被停用、删除或降级。
- **FR-041**: 系统 MUST 使用版本号或更新时间阻止并发管理覆盖。
- **FR-042**: 除登录页和健康检查外，所有应用页面 MUST 要求有效会话；静态应用壳和非敏感构建资源可以公开传输，但不得内嵌蓝图、用户或管理数据。
- **FR-043**: 系统 MUST 将 Session 绝对有效期设为 12 小时、空闲超时设为 2 小时；登录、改密和权限变化后必须轮换 Session，Session 不得超过账号 `validUntil`。
- **FR-044**: 系统 MUST 对登录实施分层限流：同一“规范化登录名 + IP”15 分钟最多 5 次失败、同一 IP 15 分钟最多 30 次失败；敏感管理写操作每管理员每分钟最多 30 次，窗口结束后自动恢复且不永久锁号。
- **FR-045**: 管理员写操作、审计事件和所需 Session 撤销 MUST 作为同一原子业务操作处理；任一环节失败必须回滚用户变更并返回可重试错误。
- **FR-046**: 用户名创建后 MUST 不可修改；管理员只能修改显示名、启用状态和有效时间，内部认证邮箱随账号创建后保持不变。
- **FR-047**: 审计事件 MUST 仅管理员可读取并保留 365 天；Session IP 与 User-Agent 仅用于安全排查并在 Session 结束后最多保留 30 天，之后删除或不可逆匿名化。
- **FR-048**: 系统 MUST 每日备份 PostgreSQL 并保留 30 天，每季度完成一次恢复演练；生产迁移必须提供回退迁移或在执行前生成可恢复备份。
- **FR-049**: API 启动时 MUST 校验 `DATABASE_URL`、至少 32 字节的 `BETTER_AUTH_SECRET` 和精确 `APP_ORIGIN`；公开健康检查只能返回存活/就绪状态，不得暴露配置、用户或数据库细节；数据库不可用时 readiness MUST 返回未就绪，受保护内容和管理 API 返回通用 `503` 且不得提供过期私有数据。
- **FR-050**: 系统 MUST 提供受保护的 `GET /api/blueprint`，支持 ETag；未登录、已失效或无权限请求不得返回任何蓝图 payload。
- **FR-051**: 精修动画图片资产 MUST 使用 `image2` 生成或重绘，并维护可拆分资产清单；图片资产不得烘焙可变数字、题干、公式或答案。
- **FR-052**: 每个完成精修的互动母题 MUST 至少包含一个与数学状态绑定的轻幽默反馈，且反馈不得遮挡关键量或替代正式解题步骤。

### Key Entities

- **KnowledgeNode**: 一个知识点，包含 K 编号、领域、学期、来源、前置知识、掌握目标和易错点。
- **Archetype**: 一个母题，包含 M 编号、主题、层级、模型、关联知识点、难度、子题和动画规格。
- **Variant**: 母题下的子题，包含参数、题干模板、解题步骤和答案规则。
- **AnimationModule**: 某个母题的交互实现注册信息，声明组件、默认状态、参数约束、步骤和资产。
- **FilterState**: 搜索词、领域、学期、层级、难度、互动和图片动画等可序列化筛选条件。
- **LearningSession**: 当前母题、子题、参数、动画步骤、答案揭示和陪练抽屉状态；第一版仅保存在当前浏览器会话。
- **UserAccount**: 可登录主体，包含登录名、显示名、角色、启用状态、生效时间、到期时间和首次改密状态。
- **AuthSession**: 服务端会话，关联用户、到期时间、创建时间和最近活动，不向前端暴露令牌值。
- **AuditEvent**: 管理操作记录，包含管理员、目标用户、动作、时间、请求标识和非敏感变更摘要。

## Success Criteria

### Measurable Outcomes

- **SC-001**: 新用户能在 10 秒内从总览找到并打开指定的 M 编号母题。
- **SC-002**: 在 `1440x900`、`1024x768`、`390x844` 三种视口中，100% 核心流程无横向滚动、文字重叠或不可达操作。
- **SC-003**: 在 Chromium 桌面基准视口、冷缓存、本地测试 API 和 500 个并发有效 Session 的设计上限内，登录提交、受保护蓝图响应和管理员单次写操作的 p95 分别不超过 1 秒；登录后的 `/learn` 首屏 2 秒内可交互；母题路由在收到蓝图响应后 300ms 内显示题目结构；五个基准动画连续操作 5 秒的平均帧率不低于 55fps。
- **SC-004**: 39 个母题的数据与现有蓝图映射完整率为 100%，117 个子题均可访问。
- **SC-005**: 所有互动母题均通过“参数联动、逐步揭示、反馈、复位、整数结果”五项自动或人工验收。
- **SC-006**: M09、M12、M20、M21、M39 五个基准动画在桌面和手机上的关键操作成功率为 100%。
- **SC-007**: 键盘用户可以不使用鼠标完成筛选、打开母题、操作步骤和关闭抽屉的完整路径。
- **SC-008**: 生产构建中蓝图 Schema、TypeScript、单元测试、组件测试和核心 Playwright 流程全部通过。
- **SC-009**: 未登录、普通用户和管理员的权限测试覆盖率为 100%，普通用户无法访问任何管理 API。
- **SC-010**: 账号停用、到期或密码重置后，下一次受保护请求在 1 秒内被拒绝，全部现有会话失效。
- **SC-011**: 管理员可在 2 分钟内完成“创建用户、设置有效期、交付临时账号”的流程。
- **SC-012**: 所有管理变更均产生可查询审计事件，敏感值泄漏数为 0。
- **SC-013**: 数据库恢复演练达到 RPO 不超过 24 小时、RTO 不超过 4 小时，且恢复后账号有效期、会话撤销状态和审计数据一致。
- **SC-014**: 每个标记为“已精修”的母题均通过大舞台、image2 分层资产、动态对象、轻幽默反馈、整数参数、减少动态和三视口无溢出的验收。

## Assumptions

- 第一版包含账号、服务端会话、用户有效期和审计日志，但不保存个人学习进度。
- 现有 JSON 蓝图和图片资产继续作为内容基线，允许在迁移时补充类型字段但不随意改写教学内容。
- 学习进度只保存在当前会话或本地浏览器；跨设备同步不在本次范围。
- 家长与孩子共用同一应用，通过陪练抽屉区分信息层级，不创建角色登录。
- 生产环境前端和 API 使用同源部署；开发环境由 Vite 代理 `/api` 到本地 API。
- 初始管理员通过部署脚本或受控命令创建，不通过公开页面注册。
- 用户使用登录名，系统为 Better Auth 内部生成不可投递的认证邮箱标识；个人邮箱不是必填资料。

## Out Of Scope

- 班级、教师后台、支付、排行榜、AI 自动批改和在线题库编辑器。
- 跨设备学习进度同步。
- 用户自助注册、社交登录、短信验证码和公开找回密码。
- 普通管理员通过 UI 创建或提升其他管理员；新增管理员通过受控运维流程完成。
- 多地域高可用、离线学习和数据库故障期间继续提供私有学习内容；故障时系统按 FR-049 明确拒绝受保护请求。
- 一次性重画全部现有图片资产；后续按动画精修批次使用 `image2` 替换不合适的整图。
- 通用动画编排器或低代码页面搭建器。
