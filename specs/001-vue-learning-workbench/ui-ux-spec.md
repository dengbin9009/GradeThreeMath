# UI/UX Specification: 主题化数学母题学习工作台

## 1. Design Direction

产品不是“39 张母题卡片的目录”，而是一张可操作的数学学习桌。当前题目的情境、关键数量和动画对象必须成为第一视觉信号；导航、筛选、路线和家长信息退到可随时取用的外围。

设计关键词：清楚、可操作、有情境、聪明好笑、不过度幼稚、家长可信。

新一轮动画升级的目标是“数学教具小剧场”：画面可以有排队、归位、亮灯、轻震和提示牌，但笑点必须服务数量关系，不能抢走题目主题。

不采用营销式 Hero、装饰性渐变球、卡片套卡片或一屏塞满统计数字。页面保持明亮中性底色，用蓝色表示当前操作、绿色表示验证通过、琥珀色表示提示、珊瑚红表示需调整；题目图片和教具承担主要色彩。

## 2. Information Architecture

### Primary Routes

| Route | Purpose | Primary Content |
|---|---|---|
| `/login` | 账号登录 | 登录名、密码、状态反馈 |
| `/learn` | 母题总览与筛选 | 搜索、筛选、推荐路径、母题结果 |
| `/learn/:archetypeId` | 母题学习 | 当前母题、默认子题、动画舞台 |
| `/learn/:archetypeId/:variantId` | 指定子题学习 | 题干、参数、动画、提示、家长话术 |
| `/account/password` | 首次改密/修改密码 | 当前密码、新密码、安全说明 |
| `/admin/users` | 管理用户 | 搜索、状态筛选、用户表格、创建用户 |
| `/admin/users/:userId` | 用户详情 | 状态、有效期、会话与审计摘要 |

筛选条件通过 query string 保存：`q`、`domain`、`term`、`layer`、`difficulty`、`interactive`、`image`。动画临时参数不写入 URL，避免分享出不可控状态。

- 左侧栏宽度 `248px`，可折叠为 `64px` 图标栏；只放知识领域、数量和已选筛选摘要。
- 顶栏高度不超过 `64px`，展示高频筛选；低频筛选进入“更多筛选”弹层。
- 主内容最大宽度不设窄文章栏，动画舞台优先获得可用宽度。
- 家长陪练和子题列表使用右侧抽屉，默认关闭，不永久占用动画宽度。
- 母题总览使用紧凑行式列表或两列列表，不使用大量漂浮卡片。
- 顶栏右侧显示用户菜单；仅管理员看到“用户管理”。账号将在 7 天内到期时显示克制的琥珀色提醒。

## 3.1 Login Layout

登录页直接展示一个清晰登录工具，不做产品营销页。桌面端表单宽度 `360-420px`，手机端占满可用宽度；品牌名称和“上海三年级数学提升”是首屏信号。

- 字段仅包含登录名、密码和显示/隐藏密码按钮。
- 允许粘贴密码并支持密码管理器。
- 提交失败使用统一错误“账号、密码或账号状态不可用”，另提供“联系管理员”提示，不暴露账号是否存在。
- 未生效或已到期账号可显示可理解状态，但不展示管理员资料或其他用户信息。
- 首次登录成功后直接进入修改临时密码页面，完成前隐藏学习导航。

## 4. Tablet And Mobile Layout

### Tablet `768-1199px`

- 左侧栏默认折叠，点击后以覆盖抽屉出现。
- 顶栏只保留搜索、筛选按钮和当前筛选数。
- 题目头、动画舞台、操作栏单列；家长陪练从右侧抽屉打开。
- 舞台最小高度 `420px`，不得缩成图片缩略图。

### Mobile `<768px`

- 数值滑块同时提供减号、数值输入、加号，保证精确整数调整。

## 5. Core Screen Anatomy

### 5.1 Problem Header

显示顺序固定：

1. 面包屑 `知识领域 > K编号 > M编号`。
2. 母题名称和少量状态标签。
3. 当前子题题干。
4. “已知 / 待求”数量条，数值使用等宽数字。

不在此区域展示功能说明、快捷键教学或大段模型定义。模型解释放在首次提示或家长抽屉中。

### 5.2 Animation Stage

- 每个动画由 `Scene + Overlay + Feedback` 三层组成。
- `Scene` 展示题目对象和真实状态；图片资产必须可独立定位。
- `Overlay` 展示尺寸线、箭头、括号、刻度和当前数量。
- `Feedback` 只在底部或对象附近出现，不使用遮挡舞台的模态框。
- 舞台具备稳定 `aspect-ratio` 或 `min-height`，动态对象不能改变外框尺寸。
- 场景内文本控制为数量标签和必要动作词，不写教学说明。
- image2 资产按图层包使用：背景、可动对象、状态帧、数学道具和反馈道具分离；图片内不写可变数字、公式或答案。
- 数量变化必须由 Vue 渲染对象数量、位置、长度、面积或分组变化，禁止只替换整张静态图。
- 每个精修动画至少有一个与数学状态绑定的轻幽默反馈，例如排错队、归位、亮提示灯或多余对象进入等待区。

### 5.2.1 Playful Feedback Rules

- 好笑但不吵：反馈时长短，位置贴近相关对象，不使用遮挡舞台的全屏庆祝。
- 错误不嘲笑：提示“哪里需要调整”，而不是把孩子的操作当笑点。
- 笑点有因果：试商太大、余数多出、括号优先、车尾未离桥、书本未回架等数学状态才能触发反馈。
- 可被关闭：减少动态模式下，反馈保留颜色、边框、提示文字和最终状态，不保留弹跳或连续位移。
- 同一屏只保留一个主要反馈，避免多个物件同时跳动。

### 5.3 Interaction Controls

- 模式切换使用分段控件。
- 二元设置使用开关或复选框。
- 数量使用步进器、整数输入或带刻度滑块。
- 步骤使用上一步/下一步和可点击步骤轨道。
- 检查、提示、复位是明确命令；复位使用图标按钮并提供 tooltip。
- 同一时刻只突出一个主操作，避免一排同权重蓝色按钮。

### 5.4 Variant And Parent Coach Drawer

抽屉包含两个标签页：

- `子题`：基础、标准、变式、挑战；显示当前题号和完成状态。
- `家长陪练`：一句讲法、常见错因、追问、完整步骤折叠区。

抽屉打开和关闭不重置动画。手机使用全宽底部抽屉，桌面使用右侧抽屉。

### 5.5 Account Menu

账号菜单包含显示名、角色、账号有效期、修改密码和退出。普通用户不显示管理入口。退出使用明确命令，完成后回到登录页并清除非必要的会话 UI 状态。

## 5.6 Admin User Management

管理员页面采用安静、紧凑、可扫描的操作界面，不沿用学习页的插画场景。

```text
┌────────────────────────────────────────────────────────────────────┐
│ 用户管理                     搜索登录名/姓名  状态  到期范围  + 添加 │
├────────────────────────────────────────────────────────────────────┤
│ 登录名   显示名   状态       生效时间       到期时间       操作      │
│ zhangsan 张三     使用中     2026-06-22     2026-09-30     ···       │
│ lisi     李四     7天内到期  2026-06-01     2026-06-29     ···       │
│ wangwu   王五     已暂停     2026-05-01     长期有效       ···       │
└────────────────────────────────────────────────────────────────────┘
```

- 顶部提供搜索、状态、到期范围筛选和“添加用户”主命令。
- 表格状态包括：未生效、使用中、即将到期、已到期、已暂停。
- 行操作使用菜单：编辑有效期、暂停/恢复、重置密码、强制下线、查看审计。
- 添加用户使用对话框或侧抽屉，字段为登录名、显示名、临时密码、生效时间、到期时间/长期有效。
- 登录名创建后只读；详情页明确区分不可修改的登录名与可修改的显示名。
- 日期时间控件显示亚洲/上海时区，并在提交前给出清晰摘要。
- 临时密码只在创建或重置成功后展示一次；提供复制按钮但不写入日志或 URL。
- 危险操作需要确认，确认内容写明用户和影响；延长有效期等可逆操作不使用多余确认。
- 手机端将表格转换为紧凑列表，每个用户仍保持状态、到期时间和操作菜单可见。
- Session 因空闲、到期或管理员操作失效时，页面保存安全的站内 redirect 并回到登录页，不保留敏感表单内容。

## 6. Filter Design

### Left Sidebar

只承载稳定分类：知识领域、知识点、当前结果数。知识领域使用图标和文本，知识点在当前领域展开，不一次显示全部 38 项。

### Top Filter Bar

高频条件：

- 搜索：名称、编号、关键词。
- 学期：上册、下册、跨学期。
- 层级：课内、课标、拔高。
- 难度：基础、提高、挑战。
- 动画：可互动、图片动画。

筛选采用即时生效，显示结果数。多个选项用菜单或复选菜单，不用一长排圆角文字按钮。已选条件以可移除标签显示在结果区顶部。

## 7. Visual System

### Color Roles

| Role | Suggested Token | Use |
|---|---|---|
| Ink | `#172033` | 标题、正文 |
| Muted | `#64748B` | 次要信息 |
| Action Blue | `#2563EB` | 当前选择、主操作 |
| Success Green | `#16845B` | 验证通过 |
| Hint Amber | `#B76600` | 提示与待观察关系 |
| Error Coral | `#D84A4A` | 非法状态与错误反馈 |
| Canvas | `#F5F7FB` | 页面底色 |
| Surface | `#FFFFFF` | 工具面与舞台 |

不使用单一蓝色系覆盖整个页面。母题场景可以引入红、黄、绿等对象色，但系统状态色含义保持稳定。

### Typography

- 中文 UI 使用系统无衬线字体栈，不加载不必要网络字体。
- 页面标题 `24-28px`，母题标题 `22-26px`，面板标题 `16-18px`，正文 `15-17px`。
- 数字使用 `font-variant-numeric: tabular-nums`。
- 字号不随视口宽度缩放；窄屏通过换行和布局调整解决。
- 字间距为 `0`，避免紧缩影响儿童阅读。

### Shape And Spacing

- 卡片和工具面圆角不超过 `8px`。
- 不嵌套卡片；页面区块使用分隔线和背景带区分。
- 图标按钮最小 `44x44px`；桌面密集工具栏可降至 `40x40px`，但需 tooltip。
- 使用 4/8px 间距体系；学习舞台内部留白优先服务对象移动。

## 8. Interaction And Motion

- 参数拖动时即时更新对象与关键量，计算结果可在拖动结束后更新，避免闪烁。
- 步骤切换动画时长 `180-450ms`；复杂情境移动不超过 `800ms`。
- 同一对象只做一种主要运动，避免同时缩放、旋转和弹跳。
- 正确反馈使用短暂高亮和对勾，不使用全屏庆祝。
- 错误反馈使用局部轻震或边框变化，并提供文字原因；减少动态模式下只改变边框和文字。
- 快速连续操作必须取消前一动画并落到最新状态。
- 幽默反馈不能改变数学答案，只能解释当前状态或引导下一步。
- 可拖动对象到达关键位置时，量线、关系式和对象状态必须同步更新。
- 所有滑杆、输入框和步进器只允许整数状态；涉及平均、除法、比例时也不能在题面或最终答案中生成小数。

## 9. Accessibility Requirements

- 所有分段控件、标签页、抽屉、提示和菜单使用 Reka UI 的语义基础组件。
- 舞台对象的视觉变化在邻近的 `aria-live="polite"` 区域给出简短状态文本。
- 拖动不是唯一操作方式；必须提供键盘或步进按钮等价操作。
- 颜色、位置和动画之外，必须有文本或数值表达当前关系。
- 焦点进入抽屉时受控，关闭后返回触发按钮。
- 图片对象使用合适 `alt`；重复计数对象使用容器级标签并将单个图片设为装饰。
- 登录错误使用 `role="alert"`，提交后焦点移动到错误摘要。
- 用户管理表格具有列标题；行操作菜单包含目标用户名，避免多个“更多”按钮名称相同。
- 日期时间输入同时提供可读文本摘要，不能只依赖日历弹层。

## 10. Reference Module Quality Bar

以下模块作为 UI 与交互基准：

| Module | Must Preserve | Vue Refactor Opportunity |
|---|---|---|
| M09 | 算筹、算盘、计算器三种工具联动 | 工具切换做成清晰分段控件，保持大舞台 |

## 11. UX Acceptance Checklist

- 打开母题后，题目和动画在首个可见屏幕内成为最大信息块。
- 筛选控件不超过顶栏一行，低频项进入菜单或抽屉。
- 任意页面不存在卡片套卡片、无意义大标题或纯装饰区域。
- 所有文本在三种目标视口内无截断、重叠和按钮溢出。
- 所有固定格式舞台和工具栏使用稳定尺寸约束。
- 精修后的母题使用大舞台 image2 分层资产，图片不退回题干旁缩略图。
- 至少一个反馈明确绑定数学状态，且不遮挡关键量。
- 家长信息默认收起，打开后不丢失孩子的动画状态。
- 每个动画都能操作、显示关键量、逐步揭示、检查和复位。
- 未登录用户不能看到学习内容；普通用户不能看到或访问管理页面。
- 用户管理页面能在桌面快速扫描，在手机完成创建、改期、暂停和强制下线。
