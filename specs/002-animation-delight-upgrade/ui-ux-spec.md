# UI/UX Specification: 精致有趣的数学动画小剧场

## Design Direction

本特性的 UI 目标是把每道母题做成“数学教具小剧场”。孩子看到的是可操作的大舞台，家长看到的是可讲解的数量关系，幽默反馈像一个轻巧的提示，而不是额外装饰。

关键词：大舞台、可拆图层、动态对象、轻幽默、整数清晰、家长可信。

## Stage Anatomy

```text
┌──────────────────────────────────────────────┐
│ 题目头：M 编号 / 标题 / 已知 / 待求          │
├──────────────────────────────────────────────┤
│                                              │
│        SceneStage 大尺寸交互舞台             │
│   背景  可动对象  数学道具  量线  反馈       │
│                                              │
├──────────────────────────────────────────────┤
│ 整数控件 / 步骤轨道 / 检查 / 提示 / 复位     │
├──────────────────────────────────────────────┤
│ 当前关系式和简短反馈                         │
└──────────────────────────────────────────────┘
```

舞台必须是详情页中最大的内容块。题干旁可以有图片动画标签，但不能把小图作为主体验。

## image2 Visual Style

- 明亮、干净、立体教具感。
- 上海小学三年级适用，亲切但不幼稚。
- 同一模块内镜头、光照、材质和比例一致。
- 背景留白足够，给可动对象和量线活动空间。
- 图片内不写题干、公式、答案、可变数字。

## Playful Feedback

允许的反馈：

- 排队、归位、亮灯、轻震、短提示牌。
- 试商太大、余数多出、括号优先、车尾未离桥、书本未回架等数学状态反馈。
- 正确完成一步时短暂高亮或盖章。

不允许的反馈：

- 全屏庆祝。
- 遮挡关系式或关键量。
- 长时间跳动、闪烁或反复弹窗。
- 嘲笑孩子的错误操作。
- 与数学无关的装饰性笑点。

## Controls

- 数值调节使用整数步进器、整数滑杆或拖动手柄。
- 步骤切换使用步骤轨道和上一/下一步。
- 模式切换使用分段控件。
- 复位使用图标按钮并带 tooltip。
- 拖动必须有键盘等价操作。

## Responsive Rules

| Viewport | Rule |
|---|---|
| Desktop | 舞台优先获得横向空间，家长陪练默认抽屉 |
| Tablet | 筛选和陪练使用覆盖抽屉，不压缩舞台 |
| Mobile | 单列布局，舞台全宽，控件在舞台下方，不遮挡安全区域 |

移动端不得把主舞台缩成题干旁缩略图。

## Reduced Motion

减少动态时：

- 对象直接到达最终位置。
- 反馈使用静态高亮、边框或文字。
- 不隐藏量线、标签和关系式。
- 不要求孩子通过运动过程才能理解结果。

## Acceptance Checklist

- 首屏最大内容块是主舞台。
- 至少一个控件会改变舞台对象状态。
- 幽默反馈贴近相关对象且不遮挡关键量。
- 图片内没有可变文字和答案。
- 三视口无横向溢出。
- 所有数字为整数。
