# 上海三年级数学提升项目 Constitution

## Core Principles

### I. 学习目标优先
每个页面、组件和动画必须服务于一个明确的数学掌握目标。题目主题、已知条件、待求量和当前学习动作必须先于筛选器、统计信息和装饰出现。不得用营销式首屏、无关装饰或大段功能说明挤占学习舞台。

### II. 数据驱动与单一事实源
知识点、母题、子题、动画规格、家长话术和难度必须来自经过 Schema 校验的结构化数据。Vue 组件不得复制题目文本或计算规则；数学计算必须由无 UI 依赖的 TypeScript 领域函数完成。母题编号 K/M 及其关系必须保持稳定。

### III. 可操作、可解释、可复位
每个互动母题必须至少支持一个直接操作、显示关键数量、分步揭示关系、明确反馈和一键复位。图片只承担情境和对象表现，数量与状态必须由程序动态渲染，不能用静态整图冒充交互。所有生成题目和中间结果必须为整数，不出现小数。

### IV. 儿童可用与家长可陪练
孩子的主要路径必须在无需阅读长说明的情况下完成；核心操作目标尺寸不得小于 44px。每个母题必须同时提供孩子提示与家长陪练话术，但家长信息默认不遮挡动画。界面用简洁中文，避免技术术语和冗余教学说明。

### V. 可访问、响应式与稳定布局
桌面、平板和手机必须提供完整学习流程。所有交互支持键盘、可见焦点、语义标签和 `prefers-reduced-motion`。动画舞台必须有稳定尺寸，动态内容不得造成页面跳动、文字重叠或控件溢出。颜色不能是唯一的信息编码方式。

### VI. 测试先行与增量迁移
领域函数和数据契约先写测试再迁移；每个母题组件必须有组件测试，核心学习路径必须有浏览器验收。Vue 重构按可独立验收的批次推进，任何阶段都不得破坏现有 39 个母题的数据、资产和计算结果。旧实现仅在对应 Vue 模块通过验收后删除。

### VII. 简单架构与明确边界
项目保持 Vue 前端、认证 API、共享契约三个明确边界，不引入通用低代码动画引擎。共享抽象只用于已经出现的重复交互模式；母题特有视觉允许独立组件。优先使用 Vue、浏览器原生动画和少量无样式无障碍组件，新增依赖必须记录理由。

### VIII. 服务端鉴权与最小权限
除登录页外，系统功能必须经过服务端会话鉴权。角色固定为 `admin` 和 `user`，授权判断不得只依赖前端隐藏。管理员只能执行规格允许的用户管理操作；用户不能访问管理接口。账号启用状态、生效时间和到期时间必须在每次受保护请求中检查。会话凭证只能存放在 `HttpOnly`、`Secure`、`SameSite` Cookie 中，不得写入 Web Storage。账号、角色、密码或有效期变化必须撤销受影响会话并记录审计事件。

## Product And Technology Constraints

- 目标用户：上海小学三年级学生及陪练家长。
- 主要语言：简体中文。
- 技术基线：Vue 3、TypeScript、Vite、Vue Router、Pinia、Reka UI、Lucide、Hono、Better Auth、PostgreSQL、Drizzle ORM、Vitest、Playwright。
- 部署形态：同源 Web 应用，由 API 服务提供鉴权、用户管理、蓝图数据和前端静态资源。
- 内容基线：保留 `data/grade3-math-blueprint.json` 的 38 个知识点、39 个母题和 117 个子题。
- 视觉基线：题目情境和动画舞台是首要视觉信号；筛选和辅助信息保持克制。
- 性能基线：首屏可交互目标小于 2 秒；切换母题后 300ms 内显示结构性内容；动画优先保持 60fps。
- 数据质量：构建时必须校验 JSON Schema、ID 引用、每个母题的子题数和整数约束。
- 安全基线：无公开注册；管理员分配账号；登录限流；密码安全哈希；严格来源校验；所有管理变更写入审计日志。

## Development Workflow And Quality Gates

1. 先更新 `spec.md`，再更新 `plan.md` 和 `tasks.md`，最后实现代码。
2. 每个任务必须可追溯到用户故事和功能需求。
3. 领域测试、组件测试、类型检查、生产构建和关键 Playwright 流程全部通过后才可完成一个迁移批次。
4. 前端改动必须在至少 `1440x900`、`1024x768`、`390x844` 三种视口检查无溢出、重叠和不可达控件。
5. M09、M12、M20、M21、M39 作为高质量交互基准，重构后不得降低其交互完整性。
6. 复杂度、依赖或范围偏离宪章时，必须在计划的 Complexity Tracking 中记录理由。
7. 鉴权、角色和有效期测试必须覆盖未登录、普通用户、管理员、未生效、已到期、已停用和会话撤销状态。

## Governance

本宪章优先于临时实现习惯。修订必须同时说明原因、影响范围和迁移策略。所有规格、计划和评审必须检查宪章合规性；不合规项不得静默进入实现。
