# 上海三年级数学提升蓝图

## 1. 定位

本蓝图面向上海小学三年级数学提升项目，内容层交付知识图谱、母题、子题和动画规格；当前网站原型已按这套蓝图覆盖 39 个母题的交互工作台。目标用户是孩子和家长：孩子通过拖动、切分、比较、试算来理解模型，家长通过讲解话术、易错点和变式路径陪练。

内容分为两层：

- 课内必会：对齐上海小学三年级沪教版数学上下册目录，并用《义务教育数学课程标准（2022年版）》第二学段要求校准。
- 提升拓展：选择三年级能理解、可视化效果强、能迁移到应用题的思维模型，如火车过桥、鸡兔同笼、植树、周期、搭配、和差倍、归一归总。

第一版默认把用户原始描述中的“字体”理解为“子题/变式题”。

## 2. 依据

- 上海 2026 年春季中小学教学用书目录确认三年级第二学期使用沪教育《数学（试用本）（三/二）》。
- 上海 2025 年秋季中小学教学用书目录补充三年级第一学期教材线索。
- 沪教版三年级上册目录覆盖：复习与提高，一位数乘两位数、三位数，一位数除两位数、三位数，年月日，米与厘米、组合图形面积、轴对称、三角形分类，植树问题、周期问题、流程图等。
- 沪教版三年级下册目录覆盖：带小括号四则运算、平方分米、组合图形面积、速度时间路程、两位数乘除、条形统计图、分数、计算器、周长、搭配、数苹果、放苹果等。

## 3. 数据接口

结构化数据位于 `data/grade3-math-blueprint.json`，供后续网站、题库、动画引擎读取。

核心接口：

```ts
type KnowledgeNode = {
  id: string;
  name: string;
  term: string;
  source: string;
  prerequisites: string[];
  masteryGoals: string[];
  mistakes: string[];
};

type Archetype = {
  id: string;
  title: string;
  layer: "课内" | "拔高";
  knowledgeIds: string[];
  model: string;
  variants: Variant[];
  animationSpec: AnimationSpec;
  parentCoach: {
    talkTrack: string;
    commonMistake: string;
    extensionPrompt: string;
  };
  difficulty: "基础" | "提高" | "拔高";
};

type Variant = {
  id: string;
  title: string;
  parameters: Record<string, string | number>;
  promptTemplate: string;
  solutionSteps: string[];
  answerRule: string;
};

type AnimationSpec = {
  scene: string;
  manipulatives: string[];
  controls: string[];
  computedValues: string[];
  revealSteps: string[];
  childFeedback: string;
};
```

JSON 中额外提供 `knowledgeToArchetypes` 反向索引，便于从知识点跳转到母题。

| 测量 | 长度单位、面积单位、面积公式、组合面积、面积估测、周长 | 课内必会 | M28-M35 |
| 图形 | 轴对称、拼嵌、多连块、三角形分类 | 课内必会 | M36, M37 |

知识点学习顺序建议：

1. 先巩固整数乘除和混合运算，再进入数量关系。
2. 速度时间路程学完后，进入火车过桥这类长度叠加模型。
3. 长方形、正方形面积掌握后，再学组合图形面积和周长面积辨析。
4. 分数先从整体分割进入，再做比较和应用。
5. 拔高模型在课内概念稳定后穿插，不把超纲题型伪装成校内必考。

## 6. 子题设计原则

每个母题至少包含 3 个子题：

1. 基础识别：判断是哪类模型，圈出关键量。
2. 标准计算：按模型公式或步骤求解。
3. 变式迁移：改变条件表达、求未知量或加入多一步条件。

拔高母题可以继续扩展到 4-6 个子题：

- 逆向求量：已知结果，反推原始条件。
- 条件缺失：判断还缺什么信息。
- 多模型综合：例如速度时间路程叠加火车过桥，面积公式叠加切割添补。

## 7. 动画设计原则

所有动画遵守同一套交互约束：

- 可拖动：核心量必须能被孩子操作，如车长、桥长、速度、份数、长宽、分割份数。
- 可计算：界面实时显示关键量和关系式，不只显示答案。
- 可揭示：默认不直接给完整解法，按“观察、建模、计算、检查”逐步揭示。
- 可复位：每个动画都提供重置和换题。
- 可讲解：家长区域显示模型名、易错点、提问句和延伸题。

第二阶段动画精修目标见 [交互动画精修路线图](animation-upgrade-roadmap.md)。本阶段把所有母题从“有交互演示”升级为“精致、有趣、轻松好笑的数学小剧场”，并统一使用 `image2` 规划可拆分图片图层。

精修动画额外遵守：

- 大舞台优先：题目图片不能只停留在题干旁的小图，核心交互必须发生在主舞台。
- 图层可拆：背景、可动对象、状态帧、量线和反馈道具分离，数量变化由程序动态渲染。
- 笑点绑定数学：幽默来自试错、排队、归位、对齐、差额和余数，不使用无关装饰抢画面。
- 图片不写答案：`image2` 资产不烘焙题干、公式、答案和可变数字，避免参数变化后画面失真。
- 对儿童友好：反馈要提示“哪里需要调整”，不嘲笑孩子，不遮挡关键量。

第一阶段高保真样板动画：

- M09 算筹/算盘/计算器：三种数字工具在同一实验台联动，保留大舞台和工具切换体验。
- M12 火车过桥/隧道：拖动车头从桥前到桥后，显示“车头到达桥头、车尾离开桥尾”两个关键时刻，路程等于车长加桥长。
- M20 鸡兔同笼：先假设全是鸡或全是兔，再用抬腿观察表现每只兔比鸡多 2 条腿，腿数差除以 2 得到兔的数量。
- M21 盈亏调整：用筹码分配、差额条和每份差展示“总差额除以每份差”的关系。
- M39 借还还原：在单一书架舞台中动态更新书架、借出篮、还回篮的书本数量，逆推和正向验算都在同一画面完成。

第二阶段优先补强样板：

- M01 连乘连除：把运算顺序做成传送带排队，错误顺序能被舞台反馈。
- M02 括号四则：把括号优先级做成运算树聚光灯。
- M03 位值乘法：把位值和进位做成可移动积木与进位小车。
- M31 组合图形面积：用切割线把 L 形、凹形图拆成长方形，或移动补块组成大长方形，显示加法和减法两条路径。

## 8. 家长陪练话术

家长讲解遵循四句话：

1. “这题在问哪几个量？”先找条件，不急着算。
2. “这些量之间是什么关系？”把文字翻译成图或式子。
3. “我们先假设/先分一份/先切一刀。”抓住模型入口。
4. “答案放回题目检查一遍。”用原题条件验算。

易错提醒：

- 看到“多、少、倍”先画线段图，不直接凑数。
- 看到“过桥、穿隧道”要判断走的是车头距离还是整列车完全通过距离。
- 看到“几分之几”必须先确定整体是谁。
- 看到“周长”和“面积”必须先问：是边线长度，还是里面大小。
- 看到统计图要先看一格代表多少，不能默认一格代表 1。

## 9. 验收清单

- 知识覆盖：三上和三下核心知识点都能在知识图谱中找到。
- 母题覆盖：共有 39 个母题，课内和拔高分层明确。
- 子题覆盖：每个母题至少 3 个子题，且包含基础、标准、变式。
- 动画覆盖：每个母题都有场景、可操作物、控件、计算量、揭示步骤和孩子反馈。
- 精修覆盖：升级后的母题必须满足大舞台、可拆 image2 图层、动态对象、数学反馈和减少动态可用。
- 家庭陪练：每个母题都有家长话术和一个明确易错点。
- 机器可读：JSON 可被标准解析器读取，并能通过字段与数量校验。
