## 2026-06-24 动画精修验收补充

新一轮逐题精修不再只验收“能显示图片”和“能切换步骤”，而是按大舞台数学小剧场验收。

| Gate | Required Evidence |
|---|---|
| 大舞台 | 首屏主视觉是交互舞台，不是题干旁边的小图；桌面、平板、手机均保留可操作空间 |
| image2 图层 | 每个精修模块记录背景、可动对象、状态帧、数学道具、反馈道具和 fallback |
| 动态对象 | 参数或步骤变化会改变对象数量、位置、长度、面积、分组或遮罩 |
| 数学反馈 | 至少一个轻幽默反馈绑定数学状态，例如试商太大、余数归位、车尾未离桥、书本未回架 |
| 整数约束 | 题干、参数控件、舞台标签、答案和关系式不出现小数 |
| 减少动态 | reduced motion 下保留最终状态、颜色、边框和文本说明 |
| 三视口 | 1280×720、834×1112、390×844 无横向溢出、遮挡和断图 |

M09、M12、M20、M21、M39 继续作为第一批基准。M01、M02、M03、M31 作为第二阶段样板，用来验证 image2 分层资产和轻幽默反馈是否能稳定复用到其余母题。

### 基准模块 image2 与陪练记录

| 模块 | image2 manifest | 资产状态 | 家长动画讲法 | 常见错因提示 |
|---|---|---|---|---|
| M09 | `apps/web/src/modules/m09/image2-manifest.ts` | 记录为 DOM 教具优先，拒收把整张工具台做成静态图的方案 | 三种工具说的是同一个位值结构 | 零所在的位不能丢 |
| M12 | `apps/web/src/modules/m12/image2-manifest.ts` | 采用现有火车过桥关键帧，补入背景、量尺、反馈和 fallback 角色 | 从车头上桥到车尾离桥，路程是车长加桥长 | 只算桥长，忘记整列车也要通过 |
| M20 | `apps/web/src/modules/m20/image2-manifest.ts` | 采用鸡兔站立和抬腿帧，记录腿数计数反馈 | 每只先少两条腿，再用剩下的腿找兔子 | 直接除总腿数，忘记每个头先贡献两条腿 |
| M21 | `apps/web/src/modules/m21/image2-manifest.ts` | 采用筹码差额关键帧，补入差额尺和分组反馈 | 先找总差额，再除以每份差额 | 盈亏一个多一个少时应合成总差 |
| M39 | `apps/web/src/modules/m39/image2-manifest.ts` | 保留独立书本 token，拒收静态整书架方案 | 倒着撤销动作，书本数量一步步回到最初 | 按题目顺序正着算，借出和还回方向反了 |

这些记录已同步到 `baselineImage2Manifests` 和 `baselineCoachNotes`，并在家长辅导抽屉中作为基准动画提示展示。
