# Animation Delight Upgrade Design

日期：2026-06-24

## Goal

把 39 个三年级数学母题的交互动画从“可用的大图演示”升级为“精致、有趣、轻松好笑的可操作数学小剧场”。本轮先完成文档和计划，不直接改实现。

## Current Baseline

- M09、M12、M20、M21、M39 已经验证了大舞台方向。
- 其余母题已经通过共享大舞台渲染器完成迁移，但很多仍偏“静态图 + 参数控件”。
- 用户明确希望后续逐个优化，不再出现题干旁边的小图片，也希望图片资产使用 `image2`。

## Design Principles

1. 数学先于好笑：笑点必须来自数学状态，比如多算、少分、试商太大、余数归位。
2. 动态对象先于整图：数量变化必须能改变对象数量、位置、长度、面积或分组。
3. 舞台先于控件：孩子先看见题目对象，再看到参数和步骤按钮。
4. 反馈不嘲笑孩子：错误反馈说明哪里不对，语气轻松，但不把孩子当作笑点。
5. 图像不写答案：image2 图片内不包含公式、答案、可变数字，避免参数变化时画面失真。
6. 家长能复述：每个动画的每一步都能对应一句简单讲法。

## Interaction Personality

产品的动画人格是“聪明的数学教具剧场”：

- 有小反应：对象会排队、归位、轻震、亮灯、举提示牌。
- 有节奏：关键步骤有 180-450ms 的轻动作，完整过桥、回架、分配等场景可到 800ms。
- 有分寸：不使用全屏庆祝、长时间弹跳、强闪烁或遮挡题目的大贴纸。
- 有一致性：同一模块内的素材镜头、光照、材质和比例统一。

## image2 Asset Design

每个模块先写 image2 资产清单，再生成图片：

- 背景：一张宽屏大舞台背景，不带题干和答案。
- 可动对象：能被 Vue 重复渲染或拖动的主体对象。
- 状态帧：表现关键动作的 2-4 个帧，例如通过、抬起、翻页、试错。
- 数学道具：量尺、括号、线段、单位格、篮子、托盘等。
- 反馈道具：提示牌、检查灯、排队牌、归位印章等。

所有可变数量由代码渲染；图片只负责对象和情境。M39 这类数量变化明显的模块，必须继续使用独立书本对象，不能退回整张书架图。

## Target Quality Bar

一个模块可以进入“已精修”状态，必须满足：

- 主舞台一眼可见题目主题。
- 至少一个主要参数可操作，并驱动场景变化。
- 至少一个步骤能把抽象关系变成对象移动或数量变化。
- 至少一个轻幽默反馈绑定到数学状态。
- 参数控件不产生小数。
- 移动端、减少动态、图片加载和断图检查通过。

## Non-goals

- 本轮不重做账号、鉴权、管理员或学习记录。
- 本轮不增加完整考试系统。
- 本轮不把所有反馈做成音效；声音以后单独评估。
- 本轮不要求一次性生成所有图片，先按母题批次推进。

## Open Design Risks

- 图片资产过多会增加首屏加载压力，需要懒加载和按模块预取。
- 图层太细会提高维护成本，需要共享舞台组件和统一 asset manifest。
- 幽默反馈容易过量，必须用验收规则限制出现位置、时长和遮挡。
