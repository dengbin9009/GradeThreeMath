# 交互动画精修路线图

更新时间：2026-06-24

本阶段目标不是再给每道题贴一张图片，而是把每个母题做成“可操作的小剧场”：孩子拖一拖、点一点、试一试，画面会跟着数学关系变，反馈带一点好笑的反应，但不会挡住题目、公式和关键数量。

本路线图的 SpecKit 跟踪文档位于 [`specs/002-animation-delight-upgrade`](../specs/002-animation-delight-upgrade/spec.md)。后续实现、验收和任务勾选以 002 文档包为准。

## 1. 升级目标

- 精致：每个舞台都有清楚的前景、中景、背景、量线和状态反馈，不再把图片缩在题干旁边。
- 有趣：核心对象可拖动、可分配、可拆合、可试错，操作结果立刻改变画面。
- 好笑：幽默来自数学状态，例如试商太大、余数多出来、括号抢先、书本排错队，而不是无关装饰。
- 可讲：动画每一步都能对应“观察、建模、计算、验算”，家长能照着画面讲。
- 可控：所有题目参数保持整数，图片不烘焙答案，关键数值由 Vue 渲染。

## 2. 统一动画契约

每个升级后的母题都必须满足这些条件：

1. 大舞台优先：图片动画占据主学习区，题干旁边只保留小图标或标签，不再把动画当缩略图。
2. 图层可拆：背景、可动对象、测量道具、反馈气泡分层，数量变化由程序增减对象，不使用一张静态整图冒充动态动画。
3. 笑点绑定数学：反馈只在孩子操作、试错、完成一步时出现，并说明当前数量关系。
4. 对象能变：拖动、滑杆、步进器和步骤切换都必须改变场景内对象位置、数量、长度、面积或分组。
5. 关键量可见：每个阶段至少显示一个当前关系式、一个对象标签或一条量线。
6. 减少动态可用：关闭动画时仍然能通过静态状态、颜色和文字看懂变化。
7. 不吵不幼稚：幽默轻巧，避免遮挡、闪烁、长时间庆祝或让孩子觉得自己被嘲笑。

## 3. image2 图片资产策略

后续图片统一用 `image2` 生成，但每个模块都按“可交互图层包”规划，而不是生成单张完整插画。

每个模块的 image2 资产包优先包含：

- `background`：无答案文字的宽屏场景背景，适配大舞台。
- `actors`：可重复摆放的对象，例如书本、筹码、积木、盒子、柱条、分割块。
- `stateFrames`：关键动作帧，例如抬腿、通过桥尾、翻日历、试商太大。
- `measureProps`：尺、量线、括号、刻度、篮子、托盘、围栏等数学道具。
- `feedbackProps`：轻量反馈物件，例如小印章、提示牌、排队牌、检查灯。
- `fallbackStill`：减少动态或图片失败时使用的一张完整静态图。

生成提示词统一遵守：

- 风格：明亮、干净、立体教具感、上海小学三年级适用、亲切但不幼稚。
- 画面：主体边缘清楚，留足移动空间，避免复杂阴影和背景噪声。
- 文本：图片内不写题目、公式、答案和可变数字；这些由界面实时渲染。
- 透明：可动对象优先用纯色背景生成后抠图，或按 image2 能力输出可分离资产。
- 一致：同一母题内所有帧保持相同镜头、光照、材质和比例。

## 7. 验收清单

每个母题升级完成后必须通过：

- 场景验收：首屏看到的是大舞台，不是题干旁边的小图。
- 资产验收：image2 资产已拆层，至少包含背景、可动对象和一种状态反馈。
- 数学验收：拖动或切步后，画面和关系式同时变化，且全程整数。
- 幽默验收：至少一个反馈与数学状态绑定，且不遮挡关键量。
- 移动端验收：390px 宽度无横向滚动，主舞台仍可操作。
- 减少动态验收：关闭动态后，状态变化仍由位置、颜色和文本表达。
- 性能验收：图片加载无断图，交互无明显卡顿，快速操作落到最新状态。
